Workflow 1: ADR Document Analysis (Core Flow)

1. Create Session → 2. Upload ADR (GCS URI) → OCR + Embed + Store → 3. Query Agent → 4. Get Response → 5. Submit Feedback

POST /api/v1/sessions with GCS URI → returns session_id. Then POST /api/v1/sessions/{id}/query to ask questions. Agent uses RAG to search embedded ADR document chunks.

Workflow 2: Policy RAG (Persistent Knowledge Base)

1. Ingest Policy → OCR + Embed → 2. List Policies → 3. Agent Queries Policies

Unlike ADR docs (session-scoped, temporary), policies are persistent and shared across all sessions. The agent has policy_search and policy_list tools to access them during queries.

Unqork HTML Rendering Rules

All HTML returned by the API is sanitized: no <script>, no event handlers, no <iframe>. CSS must be inline. Only safe elements allowed (div, p, h1-h6, ul, ol, li, table, strong, em, a, img, br, hr). The content_base64 field is the base64-encoded version for Unqork's Resolvebase64 flag.
