Test Harness Guide

Prerequisites

# 1. Start the AI agent (port 8000)
ENABLE_DEV_ROUTES=1 API_AUTH_TOKEN=dev-token-12345 VERTEX_AI_MODE=stub \
  python -m uvicorn "src.api.app:create_app" --factory --port 8000

# 2. (Optional) Start the Docker stack for legacy sources
docker compose -f docker-compose.dev.yml up
# Python mock on :8080, Java app on :8081

The harness auto-creates a demo session on load. VERTEX_AI_MODE=stub uses deterministic stub responses (no GCP credentials needed).

iframe-overlay — iframe Overlay

What it tests: Loads the ADR legacy page into #stage, then injects a fixed-position AI chat iframe in the bottom-right corner (the production integration pattern).

  1. Select iframe Overlay mode and a source
  2. Click Load ADR Page
  3. The legacy ADR page renders in #stage and the AI widget iframe appears bottom-right
  4. Test chat functionality inside the overlay iframe
  5. Verify: iframe resizes via postMessage("careconnect:height")
  6. Verify: postMessage("careconnect:close") toggles visibility

iframe-replace — iframe Replace

What it tests: Loads the legacy ADR page into #stage only — no overlay widget. Use this to verify the raw legacy page loads correctly via each source before adding the overlay.

  1. Select iframe Replace mode and a source
  2. Click Load ADR Page
  3. The legacy page renders in #stage with no AI widget attached
  4. Verify the legacy page structure and any embedded links

b64-fragment — b64 Fragment

What it tests: Simulates the Unqork Plug-In → Hidden → HTML Element pipeline. Validates our HTML passes Unqork sanitization. Closest to production Unqork flow.

  1. Select b64 Fragment mode
  2. Click Load ADR Page
  3. Purple "Additional Documentation" layout appears (left), debug panel (right)
  4. Click "Ask AI Agent" to expand the AI section
  5. Select a preset question or type a free-text question
  6. Click "Generate AI Response"
  7. Verify the right debug panel shows sanitization result: Tags/Attrs/Events stripped = 0
  8. Ask a follow-up — verify full-replace (not append) behavior

raw — Raw Debug

What it tests: Loads the legacy page into #stage with the full debug panel visible. Use to inspect raw HTML returned from any source without any widget injection.

  1. Select Raw Debug mode and a source
  2. Click Load ADR Page
  3. The legacy page HTML is rendered in #stage
  4. Inspect the source HTML in DevTools to verify structure and mount points

Source Selector

  • Live Java (:8081) — real Java app via Docker (legacy-java container)
  • Python Mock (:8080) — Python mock server via Docker (legacy container)
  • Static Fixture — /static/fixtures/clinical-viewer-display.html (no Docker needed)

Data Flow Reference

Unqork Platform
  │
  ├─ Plug-In POST /memberADR/renderHtml
  │   └─ Legacy Java app returns popup HTML
  │
  ├─ b64-fragment mode simulates:
  │   Plug-In → Hidden component → Unqork sanitize → HTML Element (full replace)
  │
  ├─ iframe-overlay mode simulates:
  │   Legacy page loads → fixed iframe widget overlays bottom-right
  │   postMessage: careconnect:height, careconnect:close
  │
  └─ Agent API
      POST /widget/v1/chat/query → {content_html, content_base64, ...}
